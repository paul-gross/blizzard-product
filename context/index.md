# Authoring conventions

The standards for writing in this repo — how the repo is laid out, and the voice its product documents are written in. Read the one file your task needs before adding or editing anything here.

| Topic | Read when… |
|-------|------------|
| [Repository structure](./structure.md) | …you are placing a new file, proposing a new folder, or checking whether something belongs in this repo at all |
| [Writing guide](./writing-guide.md) | …you are writing or reviewing any product document — a roadmap entry, a requirements doc, or a capability description |

The canon's mechanical principles apply to every file here regardless of voice: one canonical owner per fact, point don't duplicate, no retrospective framing, no manual line-wrapping — `winter-canon:/principles.md`.
